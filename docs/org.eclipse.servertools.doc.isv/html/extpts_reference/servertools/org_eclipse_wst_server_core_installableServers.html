Installable Servers

Identifier:
org.eclipse.wst.server.core.installableServers

Since:
This extension point is part of an interim API that is still under development and expected to change significantly before reaching stability. It is being made available at this early stage to solicit feedback from pioneering adopters on the understanding that any code that uses this API will almost certainly be broken (repeatedly) as the API evolves.

Description:
This extension point is used to install new server adapters from remote update manager sites.

Configuration Markup:

<!ELEMENT extension (installableServer+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>

  • point - a fully qualified identifier of the target extension point
  • id - an optional identifier of the extension instance
  • name - an optional name of the extension instance

The installableServer element is deprecated

<!ELEMENT installableServer EMPTY>

<!ATTLIST installableServer

id             CDATA #REQUIRED

name           CDATA #IMPLIED

description    CDATA #IMPLIED

vendor         CDATA #IMPLIED

version        CDATA #IMPLIED

featureId      CDATA #REQUIRED

featureVersion CDATA #REQUIRED

featureSite    CDATA #REQUIRED>

  • id - specifies a unique identifier for this extension point
  • Deprecated name - a translatable name used to identify the runtime type
  • Deprecated description - a translatable description of the runtime type
  • Deprecated vendor - a translatable vendor name
  • Deprecated version - a translatable version number
  • featureId - the id of the feature containing the server adapter bundle
  • featureVersion - the version of the feature containing the server adapter bundle
  • featureSite - the feature (update manager) site URL

Examples:
The following is an example of an installable server extension point: 

   <extension 
         point="org.eclipse.wst.server.core.installableServers">
      <installableServer
            id="com.myserver.id"
            name="My runtime name"
            description="My runtime description"
            vendor="xyz"
            version="1.0"
            featureVersion="1.0.0"
            featureId="com.myfeature"
            featureSite="xyz"/>
   </extension>
